# 평가 지표 확정

## 목적

본 실험의 목적은 텍스트 기반 RAG, 멀티모달 RAG, 상황 인지형 멀티모달 RAG의 차이를 정량적으로 비교하는 것이다.

평가는 크게 두 축으로 나눈다.

1. 검색 성능 평가: 정답 텍스트 또는 정답 이미지가 검색 결과 상위 k개 안에 포함되는지 확인한다.
2. 응답 품질 평가: 모델이 생성한 답변이 실습 안내로 적절한지 사람이 직접 평가한다.

## 평가 대상

평가에는 `SCIE용/excel/03_question_set_70.xlsx`의 70개 질의셋을 사용한다.

각 질문에는 다음 정답 정보가 포함되어 있다.

| 항목 | 의미 |
|---|---|
| 질문 번호 | 평가 질의 ID |
| 질문 | 사용자 입력 질문 |
| 정답 텍스트 | 질문에 대응되는 기준 설명 |
| 정답 이미지 | 질문에 대응되는 기준 이미지 파일명 |
| 실습 단계 | 질문이 속한 실습 맥락 |
| 질문 유형 | 질문의 성격 |

## 검색 성능 지표

### Recall@k

Recall@k는 정답 항목이 검색 결과 상위 k개 안에 포함되는지를 평가하는 지표이다.

본 연구에서는 다음 세 가지를 사용한다.

| 지표 | 의미 |
|---|---|
| Recall@1 | 검색 결과 1순위가 정답인지 평가 |
| Recall@5 | 검색 결과 상위 5개 안에 정답이 있는지 평가 |
| Recall@10 | 검색 결과 상위 10개 안에 정답이 있는지 평가 |

계산 방식은 다음과 같다.

```text
Recall@k = 정답이 Top-k 안에 포함된 질문 수 / 전체 질문 수
```

예를 들어 70개 질문 중 42개 질문에서 정답 이미지가 Top-5 안에 포함되면 이미지 Recall@5는 다음과 같다.

```text
Image Recall@5 = 42 / 70 = 0.60
```

### MRR

MRR은 정답이 몇 번째 순위에 등장했는지를 반영하는 지표이다.

정답이 1위에 있으면 1.0, 2위에 있으면 0.5, 5위에 있으면 0.2가 된다.
정답이 Top-10 안에 없으면 0으로 계산한다.

```text
MRR = 전체 질문에 대한 (1 / 정답 순위)의 평균
```

MRR은 단순히 정답 포함 여부만 보는 Recall@k보다 순위 품질을 더 잘 보여준다.

## 텍스트 검색 평가

텍스트 검색은 정답 텍스트 또는 정답 페이지와 검색된 텍스트 chunk가 얼마나 일치하는지를 기준으로 평가한다.

| 평가 | 기준 |
|---|---|
| O | 정답 텍스트와 직접적으로 같은 내용을 포함하거나, 같은 절차/설명에 해당함 |
| △ | 정답과 관련은 있지만 핵심 설명이 부족하거나 주변 내용만 검색됨 |
| X | 정답과 다른 내용이 검색됨 |

정량화가 필요할 경우 다음과 같이 점수로 변환한다.

| 평가 | 점수 |
|---|---|
| O | 1.0 |
| △ | 0.5 |
| X | 0.0 |

텍스트 검색 평가는 다음 비교군에 적용한다.

| 비교군 | 텍스트 검색 평가 |
|---|---|
| G1 키워드 기반 단순 검색 | 적용 |
| G2 텍스트 기반 RAG | 적용 |
| G3 멀티모달 RAG | 적용 |
| G4 상황 인지형 멀티모달 RAG | 적용 |

## 이미지 검색 평가

이미지 검색은 `정답 이미지` 파일명이 검색 이미지 Top-k 안에 포함되는지를 기준으로 평가한다.

| 평가 | 기준 |
|---|---|
| O | 정답 이미지가 Top-5 안에 포함됨 |
| △ | 정답 이미지가 Top-6부터 Top-10 사이에 포함됨 |
| X | 정답 이미지가 Top-10 안에 포함되지 않음 |

논문용 정량 지표는 다음을 사용한다.

| 지표 | 기준 |
|---|---|
| Image Recall@1 | 정답 이미지가 1순위인지 |
| Image Recall@5 | 정답 이미지가 상위 5개 안에 있는지 |
| Image Recall@10 | 정답 이미지가 상위 10개 안에 있는지 |
| Image MRR | 정답 이미지의 평균 역순위 |

이미지 검색 평가는 다음 비교군에 적용한다.

| 비교군 | 이미지 검색 평가 |
|---|---|
| G1 키워드 기반 단순 검색 | 제외 |
| G2 텍스트 기반 RAG | 제외 또는 참고용 |
| G3 멀티모달 RAG | 적용 |
| G4 상황 인지형 멀티모달 RAG | 적용 |

G2는 원칙적으로 이미지 검색 기능을 사용하지 않기 때문에 이미지 검색 성능 비교의 직접 대상은 아니다.
다만 텍스트 검색 결과와 연결된 이미지 후보를 별도 baseline으로 사용할 경우, 보조 실험으로만 제시한다.

## 응답 품질 평가

응답 품질은 모델이 생성한 텍스트 답변을 사람이 직접 평가한다.

평가 항목은 교수님 피드백을 반영해 다음 5개로 확정한다.

| 항목 | 평가 질문 |
|---|---|
| 정확성 | 정답 텍스트의 핵심 내용을 맞게 설명했는가? |
| 구체성 | 실습자가 바로 따라 할 수 있을 정도로 구체적인가? |
| 실습 단계 적합성 | 질문의 실습 단계와 맞는 설명을 제공했는가? |
| 안전성 | 로봇 실습에서 위험하거나 잘못된 안내를 하지 않는가? |
| 이해 용이성 | 초보 학습자가 이해하기 쉬운가? |

각 항목은 다음 기준으로 평가한다.

| 평가 | 기준 | 점수 |
|---|---|---|
| O | 충분히 적절함 | 1.0 |
| △ | 일부 적절하지만 부족함 | 0.5 |
| X | 부적절하거나 틀림 | 0.0 |

최종 응답 품질 점수는 5개 항목 점수의 평균으로 계산한다.

```text
응답 품질 점수 = (정확성 + 구체성 + 실습 단계 적합성 + 안전성 + 이해 용이성) / 5
```

## 모델 비교 지표

로컬 LLM 비교는 Qwen, Gemma, Llama를 대상으로 한다.

| 모델 | 비교 항목 |
|---|---|
| Qwen | 텍스트 답변 품질, 안정성, 응답 경향 |
| Gemma | 텍스트 답변 품질, 안정성, 응답 경향 |
| Llama | 텍스트 답변 품질, 안정성, 응답 경향 |

모델 비교에서는 이미지 검색 성능보다 텍스트 답변 품질을 중심으로 평가한다.
이미지 검색 결과는 RAG 검색 단계의 결과이므로, 동일 검색 결과를 사용한다면 모델별 차이보다 검색 구조 차이를 보는 것이 더 적절하다.

## 1차 파일럿 실험 기준

1차 파일럿에서는 모든 비교군을 완벽하게 구현하기보다, 다음 순서로 진행한다.

1. G3 현재 멀티모달 RAG의 70개 질의셋 이미지 검색 성능을 먼저 계산한다.
2. G2 텍스트 기반 RAG의 모델별 답변을 생성하고 수동 평가표에 기록한다.
3. G4 상황 인지형 re-ranking을 추가한 뒤 G3와 이미지 Recall@k, MRR을 비교한다.
4. 필요하면 G1 키워드 baseline을 추가해 단순 검색 대비 RAG의 장점을 보인다.

## 확정 결론

본 연구의 핵심 정량 지표는 Recall@1, Recall@5, Recall@10, MRR로 확정한다.
응답 품질은 정확성, 구체성, 실습 단계 적합성, 안전성, 이해 용이성의 5개 항목을 O/△/X로 평가한다.

이 구성은 교수님이 요구한 검색 성능, 응답 품질, ablation study를 모두 포함한다.
