# 비교군 확정

## 목적

SCIE 논문에서 핵심적으로 보여야 하는 것은 "시스템을 만들었다"가 아니라, 텍스트 기반 RAG와 비교했을 때 멀티모달 정보와 실습 단계 정보가 실제로 검색 성능과 응답 품질을 개선하는지 여부이다.

따라서 비교군은 다음 4개로 확정한다.

## 최종 비교군

| 구분 | 비교군 이름 | 핵심 목적 |
|---|---|---|
| G1 | 키워드 기반 단순 검색 | 전통적 검색 baseline |
| G2 | 텍스트 기반 RAG | 텍스트 임베딩 기반 RAG baseline |
| G3 | 멀티모달 RAG | 텍스트 + 이미지/도식 검색 효과 확인 |
| G4 | 상황 인지형 멀티모달 RAG | 실습 단계 정보 반영 효과 확인. 현재는 정식 구현 전 파일럿 단계 |

## G1. 키워드 기반 단순 검색

### 정의

질문 문장과 매뉴얼 텍스트 chunk 사이의 단어 일치 정도를 기반으로 관련 문서를 찾는 방식이다.

### 목적

임베딩 기반 RAG가 단순 키워드 검색보다 나은지 확인하기 위한 가장 기본적인 baseline이다.

### 사용 데이터

- `data/processed/text_chunks.json`
- `SCIE용/excel/03_question_set_70.xlsx`

### 검색 방식

1. 질문에서 주요 단어를 추출한다.
2. 각 텍스트 chunk에 질문 단어가 얼마나 포함되는지 계산한다.
3. 점수순으로 Top-k chunk를 반환한다.

### 출력

- Top-1, Top-5, Top-10 텍스트 chunk
- 해당 chunk의 heading, page
- 이미지 검색은 수행하지 않음

### 평가

- 텍스트 검색 Recall@k
- 응답 생성 평가에서는 선택적으로 제외하거나, 검색 결과 기반 단순 답변만 비교

## G2. 텍스트 기반 RAG

### 정의

BGE-M3 임베딩을 사용하여 질문과 텍스트 chunk의 의미 유사도를 계산하고, 관련 텍스트만 검색하여 LLM 답변을 생성하는 방식이다.

### 목적

멀티모달 정보를 사용하지 않는 일반 RAG baseline이다.

### 사용 데이터

- ChromaDB `doosan_manual_collection`
- BGE-M3 텍스트 임베딩
- Qwen, Gemma, Llama 4-bit LLM

### 검색 방식

1. 질문을 BGE-M3로 임베딩한다.
2. 텍스트 컬렉션에서 Top-k chunk를 검색한다.
3. 검색된 텍스트만 LLM context로 제공한다.
4. 이미지는 제공하지 않는다.

### 출력

- 텍스트 답변
- Top-k 텍스트 chunk

### 평가

- 텍스트 답변 정확성
- 구체성
- 안전성
- 이해 용이성
- 텍스트 검색 Recall@k

## G3. 멀티모달 RAG

### 정의

텍스트 검색 결과와 이미지/도식 검색 결과를 함께 제공하는 현재 시스템 구조이다.

### 목적

텍스트 기반 RAG와 비교했을 때, 이미지/도식 정보가 실습 안내에 필요한 시각 자료 검색 성능을 제공하는지 확인한다.

### 사용 데이터

- ChromaDB `doosan_manual_collection`
- ChromaDB `doosan_image_collection`
- 전처리된 이미지 폴더 `data/processed/final_refined_data`
- SigLIP 기반 이미지 필터링 및 텍스트-이미지 매핑 점수
- BGE-M3 임베딩

### 검색 방식

1. 질문을 BGE-M3로 임베딩한다.
2. 텍스트 컬렉션에서 답변용 Top-k chunk를 검색한다.
3. 텍스트 검색 결과에 연결된 이미지 후보를 수집한다.
4. 이미지 전용 컬렉션에서 관련 이미지 후보를 검색한다.
5. 페이지 근접도, 텍스트 순위, 이미지 검색 점수, SigLIP 매핑 점수를 통합해 이미지 Top-10을 정렬한다.

### 출력

- 텍스트 답변
- 검색 이미지 Top-10
- 이미지 순위
- 이미지 검색 점수

### 평가

- 이미지 Recall@1
- 이미지 Recall@5
- 이미지 Recall@10
- 이미지 MRR
- 텍스트 답변 정확성
- 이미지 적합성

## G4. 상황 인지형 멀티모달 RAG

### 정의

G3 멀티모달 RAG에 실습 단계 라벨을 추가로 반영하는 방식이다.

예를 들어 질문이 `제어기 I/O 배선` 단계에 해당하면, 해당 단계와 관련된 텍스트 chunk 또는 이미지 후보에 가중치를 부여하거나 검색 범위를 우선 제한한다.

### 목적

학습자의 현재 실습 단계 또는 상황 정보를 반영했을 때 검색 결과와 응답 품질이 개선되는지 검증한다.

### 사용 데이터

- `SCIE용/excel/03_question_set_70.xlsx`
- 각 질문의 `실습 단계` 라벨
- G3의 텍스트/이미지 검색 인덱스

### 검색 방식

1. 질문의 실습 단계 라벨을 입력값으로 사용한다.
2. 일반 멀티모달 검색을 수행한다.
3. 검색 후보 중 실습 단계와 관련된 heading, page, label, keyword가 일치하는 항목에 가중치를 부여한다.
4. 이미지와 텍스트 후보를 재정렬한다.

### 출력

- 텍스트 답변
- 상황 정보가 반영된 검색 이미지 Top-10
- 상황 정보 반영 전/후 순위 비교

### 평가

- 이미지 Recall@1/5/10 변화
- MRR 변화
- 실습 단계 적합성
- 응답 구체성
- 안전성

## 비교 구조 요약

| 항목 | G1 키워드 검색 | G2 텍스트 RAG | G3 멀티모달 RAG | G4 상황 인지형 멀티모달 RAG |
|---|---|---|---|---|
| 텍스트 검색 | O | O | O | O |
| 임베딩 검색 | X | O | O | O |
| LLM 답변 | 선택 | O | O | O |
| 이미지 검색 | X | X | O | O |
| 실습 단계 정보 | X | X | X | O |
| 목적 | 단순 검색 기준선 | 일반 RAG 기준선 | 이미지/도식 효과 확인 | 상황 정보 효과 확인 |

## Ablation Study 구성

SCIE 논문에서는 다음 순서로 ablation study를 제시한다.

| 실험 | 구성 | 확인하려는 효과 |
|---|---|---|
| A1 | 키워드 검색 | 단순 검색 기준 성능 |
| A2 | BGE-M3 텍스트 RAG | 의미 기반 텍스트 검색 효과 |
| A3 | A2 + 이미지 전용 컬렉션 | 이미지/도식 검색 추가 효과 |
| A4 | A3 + 실습 단계 라벨 | 상황 정보 반영 효과 |

## 우선 구현 순서

1. G1 키워드 검색 평가 스크립트 작성
2. G2 텍스트 기반 RAG 평가 스크립트 정리
3. G3 현재 멀티모달 RAG를 70개 질의셋 기준으로 재평가
4. G4 상황 단계 라벨 기반 re-ranking 방식 추가
5. 네 비교군의 Recall@k, MRR, 응답 품질 평가표 생성

## 현재 구현 상태

| 비교군 | 구현 상태 | 비고 |
|---|---|---|
| G1 키워드 검색 | 미구현 | 간단한 token overlap 또는 BM25 방식 필요 |
| G2 텍스트 RAG | 부분 구현 | 현재 앱의 텍스트 검색 경로 활용 가능 |
| G3 멀티모달 RAG | 구현 완료 | `src/rag_search.py` 기준 |
| G4 상황 인지형 멀티모달 RAG | 미구현 | 단순 실습 단계 라벨 파일럿만 수행. 정식 구현에는 단계별 page/section/keyword 매핑표 필요 |
