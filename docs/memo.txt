✅ 수정 1 — embedding_text_image.py (약점 2번 해결)
----------------------------------------------------------------

📌 어떤 문제였나요?
  "텍스트와 연관된 이미지를 찾을 때, 관련 없는 이미지까지 전부 저장하고 있었어요."

  예를 들어, 같은 페이지에 이미지가 10개 있으면
  가장 가까운 이미지 1개만 골라야 하는데
  10개를 다 저장하고 있었던 거예요.
  그래서 나중에 화면에 엉뚱한 그림이 뜰 수 있었습니다.

🔧 어떻게 고쳤나요?
  두 가지 조건을 추가했습니다.

  조건 1. 거리 제한 (MAX_DISTANCE = 300)
    → 텍스트와 이미지 사이의 거리가 300보다 멀면
      "관련 없는 이미지다" 판단하고 아예 제외합니다.
    → 300이라는 숫자는 PDF 페이지 좌표 기준이며, 필요하면 조절 가능합니다.

  조건 2. 최대 개수 제한 (TOP_N_IMAGES = 2)
    → 거리 조건을 통과한 이미지 중에서도 최대 2개만 저장합니다.

📂 수정된 코드 위치: embedding_text_image.py
  - 37~39번 줄: MAX_DISTANCE, TOP_N_IMAGES 상수 추가
  - 76~83번 줄: 정렬 → 거리 필터링 → 상위 2개 선택 로직으로 교체

⚠️ 이 파일을 다시 실행하면 rag_db 폴더를 새로 만들어야 합니다.
   (기존 rag_db 폴더를 삭제한 뒤 실행하세요)

✅ 수정 2 — app_qwen.py / app_gemma.py / app_llama.py (약점 3번 해결)
----------------------------------------------------------------

📌 어떤 문제였나요?
  "검색 결과 1~5위 문서를 찾아도, 이미지는 무조건 1위 문서 것만 보여줬어요."

  1위 문서에 연관 이미지가 없으면
  2~5위에 딱 맞는 도면이 있어도 무시하고
  "이미지를 찾을 수 없습니다"라고 표시했습니다.

🔧 어떻게 고쳤나요?
  1위부터 5위까지 문서를 전부 돌아보면서 이미지를 모읍니다.
  같은 이미지가 중복으로 나오면 한 번만 보여줍니다.
  각 이미지 아래에 출처(몇 순위 문서, 어떤 섹션, 몇 페이지)를 표시합니다.

  예시:
    기존 caption → "1순위 관련 기술 도식"
    수정 caption → "[3순위] 서보 모터 구조 (Page 115)"

📂 수정된 코드 위치: app_qwen.py / app_gemma.py / app_llama.py
  - 91~104번 줄: Top-5 전체 순회하며 이미지 수집 + 중복 제거 로직 추가
  - 124~130번 줄: 이미지 출처 정보를 caption에 함께 표시

✅ 수정 3 — text_filter.py (약점 4번 해결)
----------------------------------------------------------------

📌 어떤 문제였나요?
  "텍스트를 여러 조각으로 나눌 때, 위치 정보(bbox)를 잘못 물려줬어요."

  좀 더 쉽게 설명하면:
  원본 문단이 문장 10개짜리라면, 이 문단에는 위치 정보도 50개가 있어요.
  이걸 의미별로 3토막 냈을 때,
    - 1토막(문장 1~4): 위치 정보가 1~20번이어야 맞는데 → 전체 50개를 줬음 (잘못됨)
    - 2토막(문장 5~7): 위치 정보가 21~35번이어야 맞는데 → 전체 50개를 줬음 (잘못됨)
    - 3토막(문장 8~10): 위치 정보가 36~50번이어야 맞는데 → 전체 50개를 줬음 (잘못됨)

  그러면 나중에 이미지 매핑할 때 엉뚱한 위치를 기준으로 계산하게 됩니다.

🔧 어떻게 고쳤나요?
  각 문장이 PDF 어느 위치에 있는지 추적하는 기능을 추가했습니다.
  문단을 자를 때 그 조각에 포함된 문장의 위치만 정확하게 물려줍니다.
  혹시 위치 정보를 못 찾으면, 안전하게 전체 위치를 쓰도록 했습니다(폴백).

📂 수정된 코드 위치: text_filter.py
  - 18번 줄: current_span_bbox_pairs 리스트 추가 (문장↔위치 쌍 기록용)
  - 40~67번 줄: bbox_record 분리 + span_bbox_pairs 누적 저장
  - 82~91번 줄: find_bboxes_for_sentence() 함수 새로 추가
  - 95~158번 줄: semantic_chunking() 내부 로직 전면 수정
  - 177~185번 줄: JSON 저장 시 내부용 데이터(span_bbox_pairs) 제거

⚠️ 이 파일을 다시 실행하면 text_chunks.json이 새로 만들어집니다.
   그 후 embedding_text_image.py도 다시 실행해야 합니다.
